# InboxDock 统一收集入口与自动收边设计

## 背景

当前版本把文件拖放事件绑定在收集页顶部的固定小块上，用户必须精确拖到该区域才能创建暂存材料。草稿输入框只处理文字和链接，没有处理剪贴板图片；文件材料也没有备注字段。原有“收起”行为会把窗口缩成 52×52 方块，需要手动展开和收起，不符合常驻桌面工具应当低干扰、随用随出的体验。

“今日”页使用自定义 `ComboBox` 模板显示 `SelectionBoxItem`，选中项没有应用 `DisplayMemberPath`，因此界面显示 `CategoryOption { Value = ..., Label = ... }` 对象字符串，而不是分类名称。

本轮改造统一解决材料入口、文件备注、多文件确认、输入排版、今日分类显示、自动收边动画和常驻内存占用问题。

## 目标

- 删除顶部固定拖放小块，让整个收集页面都能接收文件。
- 支持从剪贴板粘贴图片、复制的文件、网页链接和普通文字。
- 同一次拖入的多个文件组成一个材料组，共享备注并一次确认写入 Inbox。
- 允许文件材料在确认前追加或修改备注。
- 修正底部文字编辑器的光标、内边距、行高和快捷键提示排版。
- 修复“今日”页选中分类显示为对象代码的问题。
- 用自动滑入屏幕边缘的箭头把手替代原来的 52×52 折叠方块。
- 展开状态连续 10 秒无用户操作后自动收边；鼠标滑到箭头处自动展开并重新计时。
- 保持四边吸附、多显示器和缩放比例支持。
- 降低常驻状态下不必要的视觉树、集合和资源占用。

## 非目标

- 不增加 OCR、图片内容识别或 AI 自动分类。
- 不改变现有 Vault、Inbox、Daily 和附件目录结构。
- 不增加跨设备同步或网络上传。
- 不提供多个独立暂存卡片的批量勾选界面；“一键收进 Inbox”指同一次拖入形成的文件组一次确认。
- 不全面重写应用为纯 MVVM，也不改造无关设置流程。

## 方案选择

采用“统一材料入口 + 独立自动收边控制器”的方案。

局部事件补丁虽然改动较少，但会继续把拖放、剪贴板、闲置计时和窗口动画堆积在 `MainWindow` 中。全面重写架构会扩大本轮风险。推荐方案把可测试的材料识别、备注写入和窗口状态计算下沉到独立组件，WPF 窗口只负责事件转发和视觉呈现。

## 统一材料入口

### 拖放范围

删除收集页顶部的 `DropZone`。收集页根容器设置 `AllowDrop=True`，处理 `DragEnter`、`DragOver`、`DragLeave` 和 `Drop`。

文件进入收集页时，在现有内容上方显示不占布局的半透明拖放遮罩。遮罩使用虚线或高亮边框和“松开以加入材料桶”提示，不要求用户瞄准特定区域。离开或完成拖放后遮罩立即消失。

目录继续拒绝，并在底部状态栏显示明确错误。一个拖放操作中的所有普通文件作为一个文件材料组暂存。

### 剪贴板识别顺序

在草稿编辑器获得焦点并收到 `Ctrl+V` 时，按以下顺序处理：

1. 剪贴板文件列表：作为一个文件材料组。
2. 剪贴板位图：编码为 PNG 后作为单文件材料组。
3. 单个绝对 `http` 或 `https` 地址：作为链接材料。
4. 其他文字：插入当前光标位置，保留原有选择区替换行为。

该顺序避免复制文件时误读成路径文字，也避免图片剪贴板同时带有说明文字时重复创建材料。

### 剪贴板图片暂存

新增图片暂存入口，将 WPF `BitmapSource` 以 PNG 编码写入 InboxDock 自有的暂存目录。文件名使用本地时间和短随机标识，例如 `clipboard-20260717-203015-a1b2c3.png`。

图片写入临时文件完成后再加入暂存快照。失败时删除临时文件，不创建卡片，并在状态栏显示实际错误。原始剪贴板内容不受影响。

## 文件组与备注

`StagedMaterial` 新增可空的 `Note` 字段。旧版 `staging.json` 没有该字段时按空字符串兼容读取，不需要迁移脚本。

同一次拖入或粘贴的多个文件保持为一个 `Files` 类型材料：

- 卡片标题显示首个文件名；多于一个文件时附加“等 N 个文件”。
- 摘要显示文件数量和总大小。
- 确认面板提供“备注（可选）”编辑器。
- 备注在输入时保存在内存中，失焦或短暂防抖后写入暂存 JSON。
- 点击“收进 Inbox”前强制保存最新备注，避免快速输入后立即确认造成丢失。

`InboxCaptureService.CaptureFilesAsync` 接收可选备注。生成的 Markdown 在附件列表之前写入备注正文；没有备注时保持现有输出格式，避免无意义空段落。

一次确认会复制整个文件组的所有暂存文件、生成一篇 Inbox 笔记，并在成功后删除该组的 InboxDock 暂存副本。任何一个文件失败时保留整张卡片及备注，便于重试。

## 收集页布局

删除顶部 47 像素拖放区后，主要空间分为材料列表和底部草稿编辑器。

草稿编辑器调整为：

- 最小高度约 58，最大高度约 112。
- 顶部对齐，统一 10–12 像素内边距。
- 明确行高，中文输入和多行光标位置稳定。
- `Ctrl+Enter` 提示放在编辑器外侧的独立底栏，不覆盖文字或光标。
- Enter 正常换行，Ctrl+Enter 创建文字材料。
- 内容增长时在最大高度前适度扩展，超过后出现滚动条。

材料列表改用支持 UI 虚拟化的列表控件，避免卡片数量增加时一次创建全部视觉元素。拖放遮罩、确认面板和删除确认继续作为覆盖层，不改变主布局尺寸。

## 今日页修复

分类下拉框不再依赖 `SelectionBoxItem` 的默认对象显示。模板显式绑定选中 `CategoryOption.Label`，下拉项也使用同一显示模板。

验收时选中框和下拉列表都只能出现：

- 完成
- 学习
- 问题
- 灵感

不会再出现类型名、属性名、花括号或对象调试字符串。同步整理选中项前景色、焦点边框和下拉项高亮，确保绿色按钮上的文字保持可读。

## 自动收边状态模型

窗口状态调整为：

- `Expanded`：完整窗口可交互。
- `Peeking`：窗口主体滑出屏幕，仅保留小箭头把手。
- `Animating`：展开或收边过渡中，阻止重复动画。

不再使用旧的 52×52 `Collapsed` 视觉和手动折叠逻辑。标题栏原“收起”按钮改为立即收边，方便用户主动隐藏；自动收边使用同一流程。

### 闲置计时

进入 `Expanded` 后启动 10 秒倒计时。以下事件都重置倒计时：

- 鼠标移动、点击或滚轮。
- 键盘输入、文本选择和粘贴。
- 文件拖入、拖动经过和释放。
- 切换标签、选择分类、点击卡片或按钮。
- 窗口重新展开或获得焦点。

以下状态暂停自动收边：

- 草稿或备注编辑器正在进行输入法组合输入。
- 文件正在复制到暂存区或 Inbox。
- 确认面板、删除确认或设置目录对话框处于打开状态。
- 鼠标仍位于窗口内部。

当暂停条件解除后重新开始完整 10 秒，而不是继续使用旧的剩余时间。这样不会在用户刚结束操作时立即收边。

### 箭头把手

`Peeking` 状态只保留约 16–22 像素宽、40–48 像素高的把手，具体宽高按停靠边缘旋转。箭头始终指向屏幕内部：左边缘向右、右边缘向左、顶部向下、底部向上。

鼠标进入把手立即触发展开。展开完成后重新启动 10 秒倒计时。鼠标短暂越过窗口边界不立即隐藏，由闲置计时统一决定。

### 动画

展开和收边同时动画化 `Left`、`Top`、`Width`、`Height`、主体透明度和轻微缩放，目标时长约 240–280ms。使用平滑的 ease-out/ease-in-out 曲线，不使用弹跳或过冲。

收边时窗口朝当前停靠边缘移动，主体逐渐缩小并淡出，把手保持清晰；展开时从把手位置向屏幕内部展开。关闭系统动画时直接切换到最终几何状态。

动画完成后清除 WPF 动画值并写入最终基础值，避免后续拖动、DPI 变化和多显示器切换受到残留动画影响。

## 窗口几何与持久化

保留四边最近边缘计算和向屏幕内部展开规则。窗口状态文件新增显示模式字段，并兼容旧的 `Collapsed` 值：旧折叠状态在首次启动时转换为 `Peeking`。

持久化以下内容：

- 停靠边缘。
- 沿边缘的位置。
- 上次是否处于 `Peeking`。
- 置顶状态。

恢复时始终根据当前显示器工作区重新夹紧位置。显示器断开、任务栏位置变化或 DPI 改变后，箭头和展开窗口都必须保持可见。

## 内存与资源优化

优化集中在常驻场景的可验证问题，不使用强制 `GC.Collect`：

- 材料卡片列表启用 UI 虚拟化和容器回收。
- “今日”页内容延迟创建，未访问时不实例化完整控件树。
- `Peeking` 状态隐藏并卸载主体内容的重型效果，展开时恢复。
- 共享画刷、几何和可冻结资源调用 `Freeze` 或保留为静态资源。
- 自动收边使用单个可停止的 `DispatcherTimer`，窗口隐藏或退出时停止。
- 退出时释放 `NotifyIcon`、菜单、计时器和可取消令牌源。
- 草稿与备注保存共用防抖策略，取消并释放旧令牌，不为每个按键保留长期任务。
- 暂存材料加入后直接更新内存快照，避免为了读取刚写入的数据重复反序列化整个 `staging.json`。
- 避免在每次刷新时清空并重建所有卡片；对新增、更新和删除使用最小集合变更。

内存验收以同一 Release 构建、相同暂存数据和相同等待时间进行前后对比。WPF/.NET 基线内存受运行时影响，因此目标是消除持续增长和明显无用占用，而不是承诺固定 MB 数值。

## 组件边界

### `ClipboardMaterialReader`

读取剪贴板并返回文件、图片、链接、普通文字或空结果。它不直接修改 UI，也不写 Vault。

### `MaterialStagingService`

负责创建文件组、图片材料、文字材料和链接材料，更新备注并维护内存快照。文件系统写入通过现有暂存服务完成。

### `StagedCaptureService`

确认前使用材料的最新备注调用 Inbox 捕获服务。成功才移除暂存材料；失败保留材料和错误信息。

### `AutoPeekController`

维护闲置计时、暂停原因和 `Expanded/Peeking/Animating` 状态。窗口只把用户活动通知给控制器，并执行控制器请求的动画。

### `WindowDockCalculator`

继续负责纯几何计算，并增加箭头把手目标矩形和屏幕外偏移的测试用计算，不依赖 WPF 控件。

## 错误处理

- 剪贴板被其他程序占用时不崩溃，状态栏提示稍后重试。
- PNG 编码或暂存写入失败时不创建卡片并清理临时文件。
- 拖入目录时拒绝整次操作并说明不支持文件夹。
- 文件组捕获失败时保留全部暂存文件、备注和错误状态。
- 备注保存失败时禁止立即确认，先提示用户重试，避免备注静默丢失。
- 动画期间收到新的展开或收边请求时合并为最终目标，不叠加多个 Storyboard。
- 自动收边控制器异常不影响材料保存；最坏情况下窗口保持展开。

## 测试策略

### 单元测试

- 剪贴板内容优先级：文件、图片、链接、普通文字。
- 多文件一次输入生成一个材料组。
- 旧暂存 JSON 缺少 `Note` 时正常加载。
- 更新备注后 JSON 往返保持内容。
- 文件 Markdown 有备注和无备注两种格式。
- 今日分类选中项显示标签而非对象字符串。
- 10 秒活动重置、暂停和恢复规则。
- 四条边的箭头矩形和向内展开矩形。
- 动画目标请求合并，不产生重复状态切换。

### 集成测试

- 暂存剪贴板图片并确认进入附件目录和 Inbox 笔记。
- 同时暂存多个文件、添加共享备注、一次确认全部写入。
- 捕获失败后文件组和备注仍可恢复和重试。
- 应用重启后文件备注仍存在。
- 旧版 `staging.json` 和 `window-state.json` 兼容加载。

### 桌面手工验证

- 收集页任意空白和列表区域都能拖入文件。
- 拖入时遮罩出现，离开后消失。
- 从截图工具、浏览器和资源管理器复制图片后可直接粘贴。
- 输入框中英文输入、换行、选择文本和光标位置正常。
- 今日分类框不再显示 `CategoryOption`。
- 左、右、上、下四边闲置 10 秒后均能平滑收边。
- 鼠标进入箭头后立即展开，持续操作时不会误收边。
- 100%、125%、150% DPI 和多显示器下位置正确。
- Release 版长时间展开/收边循环后句柄数和私有内存不持续增长。

## 验收标准

- 顶部固定拖放小块已删除。
- 用户无需瞄准小块即可把一个或多个文件拖入收集页。
- 剪贴板图片可粘贴并作为 PNG 材料收进 Inbox。
- 同一次输入的多个文件仅需一次确认，并可附加一条共享备注。
- 文件备注写入生成的 Inbox Markdown，重启前后不丢失。
- 底部编辑器文字、光标和快捷键提示互不遮挡。
- 今日页只显示分类标签，不显示对象代码。
- 展开后无操作 10 秒自动滑入当前屏幕边缘，仅留下箭头。
- 鼠标进入箭头后平滑展开，并开始新的 10 秒循环。
- 输入、拖放、确认和捕获过程中不会自动收边。
- 全部自动化测试和 Release 构建通过。
- 对比基线时不存在因计时器、动画、卡片容器或事件订阅造成的持续内存增长。
