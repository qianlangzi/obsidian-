# 变更日志

本文件记录 InboxDock 所有公开版本的变更。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.3.0] - 2026-07-24

### 新增

- **可配置收集目标**：支持追加到固定文件、追加到周期文件（如 Daily）、在目录创建新笔记和只暂存四种写入方式
- **两步首次配置**：新用户只需选择 Vault 和默认保存方式即可开始使用
- **目标管理**：设置界面支持新建、编辑、删除、设为默认和排序收集目标
- **预览确认**：首次使用或修改后的目标写入前展示 Markdown、路径和原因
- **批量处理**：多选卡片批量保存到指定目标或批量移除
- **状态挂件**：贴边把手显示待处理数量（99+ 截断），列表汇总条显示总数、最早时间和失败指示
- **可配置自动收边**：支持 5 秒、10 秒、30 秒和永不自动收回
- **图钉状态**：置顶窗口时暂停自动收边
- **写入成功短延时收边**：成功后 2 秒收回，用户输入可取消
- **全局快捷键**：默认 `Ctrl+Shift+Space` 呼出，设置界面捕获真实按键组合
- **单实例**：重复启动复用现有实例并呼出窗口
- **开机自启动**：设置中可开启、关闭，便携版移动后提示修复
- **完善反馈**：成功显示目标名称和打开笔记按钮，失败显示"材料仍安全"并支持复制错误
- **隐私安全日志**：滚动日志（256KB 上限），不记录笔记正文和文件内容
- **诊断信息**：设置中可复制版本、系统、Vault 状态诊断，路径已遮蔽
- **GitHub 更新检查**：启动后非阻塞检查最新发布，有更新时显示徽章
- **安全设置迁移**：旧配置自动迁移到 v2，迁移前备份，失败可回滚

### 改进

- 拖入文件先复制到暂存区再展示卡片，原文件不被移动或删除
- 写入失败时保留卡片并重新打开确认面板，允许重试或换目标
- 撤销只影响本次写入标记区域，用户修改后拒绝危险撤销
- 路径校验确保笔记和附件始终位于 Vault 内
- 同名文件自动增加编号，不覆盖已有文件

### 安装与发布

- 提供 Inno Setup 安装包（`InboxDock-Setup-<version>.exe`）
- 提供便携 ZIP（`InboxDock-<version>-portable-win-x64.zip`）
- 提供 SHA256 校验文件
- GitHub Actions `v*` 标签自动构建、测试、打包并发布
